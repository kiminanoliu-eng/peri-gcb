## 核心教训

### 7个核心原则

1. **质量优先于速度** - 慢即是快，做对一次比返工多次更快
2. **验证正确性，不只是有效性** - HTTP 200 ≠ 内容正确
3. **不要依赖工具的不完整输出** - WebFetch可能返回不完整结果
4. **严格遵守命名规则** - 文件名必须与slug完全匹配
5. **只做用户明确要求的工作** - 不擅自修改已完成的产品
6. **数据来源必须准确** - 项目必须从产品页面HTML提取
7. **在正确的地方搜索** - 在PERI频道内搜索，不在YouTube主页

### 三步验证法

适用于PDF链接、项目案例、YouTube视频：

1. **有效性验证**: 链接/资源是否存在（HTTP 200）
2. **类型验证**: 是否是正确的类型（PDF/项目/视频）
3. **内容验证**: 内容是否是该产品的（人工确认）

## 标准工作流程（30分钟/产品）

### 阶段0: 开始前（5分钟）
```bash
# 1. 确认slug
grep -i "产品名称" products_v2.json

# 2. 预检slug
python3 pre_flight_check.py {slug} slug
```

### 阶段1-4: 数据收集（15分钟）
- 提取产品信息（curl获取HTML）
- 提取项目案例（从HTML，不用WebFetch）
- 搜索YouTube视频（在PERI频道内）
- 查找PDF链接（三步验证法）

### 阶段5: 创建JSON（5分钟）
```bash
# 创建文件: {slug}_complete.json
python3 pre_flight_check.py {slug} json
python3 verify_product.py {slug}
```

### 阶段6-8: 生成和部署（5分钟）
```bash
python3 rebuild_site_v2.py
python3 pre_flight_check.py {slug} all
bash deploy-peri-gcb.command
```

## 关键检查点

### 最重要的3个检查点

1. **文件命名** - 创建文件前运行 `pre_flight_check.py {slug} slug`
2. **PDF内容** - 下载并打开PDF，人工确认是该产品的手册
3. **项目来源** - 从产品页面HTML提取，不从china_projects.json选择

### 必须人工确认的3项

1. PDF内容是该产品的手册
2. YouTube视频是该产品的英语介绍
3. 项目案例在该产品的cn.peri.com页面上
